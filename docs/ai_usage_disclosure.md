# AI Usage Disclosure

**AAI-510 Agentic AI Systems — Final Team Project**
University of San Diego, MS Applied Artificial Intelligence
**Project:** SOC Triage Agent for NovaPay Financial
**Date:** June 17, 2026

---

This document discloses all uses of generative AI tools by each team member during the development of this project. For each tool, we describe what it was used for, which project artifacts it contributed to, and how the team verified and validated the AI-generated output.

---

## Marquise Oliver — Product Manager

### 1. Grammarly Writing Assistant

| Detail | Description |
|--------|-------------|
| **Tool** | Grammarly Writing Assistant |
| **Purpose** | Proofreading and copy-editing recommendations |
| **Artifacts Touched** | `README.md`, `docs/business_case.md` |
| **How It Was Used** | Grammarly was used to review drafted prose for grammar, spelling, punctuation, clarity, and conciseness. Suggested revisions were reviewed individually and accepted, modified, or rejected based on the author's judgment. |
| **Verification** | All Grammarly suggestions were reviewed manually before acceptance. The author retained full editorial control over tone, technical accuracy, and content. No substantive claims, data, or analysis were generated by Grammarly. |

### 2. Google Gemini (Image Generation)

| Detail | Description |
|--------|-------------|
| **Tool** | Google Gemini |
| **Purpose** | Generation of the NovaPay business logo and supporting graphs/charts |
| **Artifacts Touched** | `docs/images/novapay_logo.png`, visual assets embedded in `docs/business_case.md` |
| **How It Was Used** | Google Gemini was used to generate the NovaPay Financial business logo used in the README header and project branding. It was also used to create data visualization graphics (charts and graphs) illustrating ROI projections, cost comparisons, and operational metrics for the business case document. |
| **Verification** | All generated images were reviewed for accuracy, visual clarity, and alignment with the project's data and narrative. Charts and graphs were cross-checked against the underlying calculations in `docs/roi_calculation.md` to ensure the visuals faithfully represented the quantitative analysis. The logo was reviewed by the team for appropriateness and branding consistency. |

### 3. Microsoft Copilot (Presentation Design)

| Detail | Description |
|--------|-------------|
| **Tool** | Microsoft Copilot |
| **Purpose** | Final presentation video theme layout and slide design recommendations |
| **Artifacts Touched** | Final presentation slide deck and video assets |
| **How It Was Used** | Microsoft Copilot was used to create the visual theme and layout for the final presentation video. It also provided recommendations to improve slide layout, sequencing, and visual flow to ensure a professional and coherent narrative structure across the presentation sections. |
| **Verification** | All Copilot-generated layouts and recommendations were reviewed by the Product Manager. Slide content, data, and technical claims were authored and verified by team members independently. Copilot was used solely for visual design assistance and structural suggestions — not for generating substantive technical or analytical content. |

## Integrity Statement

All substantive technical work — including system architecture, agent design, data pipeline engineering, evaluation methodology, code implementation, and analytical conclusions — was performed by the team members listed above. AI tools were used in supporting roles (proofreading, visual asset generation, presentation design) and did not replace human judgment, technical decision-making, or original analysis. All AI-generated outputs were reviewed and validated before inclusion in project deliverables.
